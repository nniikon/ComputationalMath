# Подробное пояснение к лабораторной работе

В папке `PDE/` лежит отчет `pde_lab_report.ipynb`. Это основной Jupyter Notebook: в нем есть постановки задач, формулы, код, графики, таблицы ошибок и выводы. Этот файл объясняет суть лабораторной более подробно, чтобы было понятно, что именно происходит математически и численно.

Лабораторная состоит из трех разных типов уравнений в частных производных:

- гиперболическое уравнение: одномерный перенос;
- параболическое уравнение: нестационарная теплопроводность;
- эллиптическое уравнение: стационарная теплопроводность.

Эти три класса ведут себя по-разному, поэтому для них используются разные численные методы.

## Общая идея численного решения PDE

Уравнение в частных производных задает связь между значениями функции и ее производными. Компьютер не работает с непрерывной областью, поэтому область заменяется сеткой.

Для одномерной задачи используются точки

```tex
x_i = ih, \quad h = 1/N.
```

Для двумерной задачи используются узлы

```tex
x_i = ih_x,\quad y_j = jh_y.
```

Непрерывная функция `u(x,t)` или `u(x,y,t)` заменяется набором чисел:

```tex
u_i^n \approx u(x_i,t_n),
```

или

```tex
u_{i,j}^n \approx u(x_i,y_j,t_n).
```

Производные заменяются конечными разностями. Например:

```tex
u_x \approx (u_i - u_{i-1}) / h,
```

```tex
u_{xx} \approx (u_{i-1} - 2u_i + u_{i+1}) / h^2.
```

После такой замены дифференциальное уравнение превращается в систему алгебраических формул, по которым можно считать значения в узлах сетки.

## 1. Линейное одномерное уравнение переноса

### Постановка

Решается уравнение

```tex
u_t + c u_x = 0.
```

В отчете взято `c = 1`. Физический смысл такой: начальный профиль движется вправо с постоянной скоростью `c`, не меняя своей формы. Если в начальный момент был прямоугольный импульс, то точное решение через время `t` это тот же прямоугольный импульс, просто сдвинутый вправо на `ct`.

В отчете используется периодическая область `[0,1]`. Это значит: если профиль вышел через правую границу, он появляется слева. Такой выбор удобен, потому что точное решение известно:

```tex
u(x,t) = u_0((x - ct) mod 1).
```

Это точное решение используется для вычисления ошибок.

### Число Куранта

Для уравнения переноса важна величина

```tex
sigma = c tau / h,
```

где `tau` это шаг по времени, а `h` шаг по координате. Она показывает, на какую долю ячейки переносится информация за один временной шаг.

Если `sigma` слишком большое, явные схемы могут стать неустойчивыми. В отчете используется значение около `0.45`, то есть меньше единицы.

### Схема CIR

CIR, или схема Куранта--Изаксона--Риса, для `c > 0` записывается так:

```tex
u_i^{n+1} = u_i^n - sigma (u_i^n - u_{i-1}^n).
```

Здесь используется левый сосед, потому что при `c > 0` информация идет слева направо. Это называется разностью против потока.

Плюсы:

- схема простая;
- устойчивая при `0 <= sigma <= 1`;
- не дает сильных осцилляций около разрывов.

Минус:

- схема сильно размазывает фронты, потому что имеет численную диффузию.

На графике это видно так: прямоугольный импульс после расчета становится не таким резким, его края сглаживаются.

### Схема Мак-Кормака

Схема Мак-Кормака это предиктор-корректор. Сначала делается прогноз, затем коррекция:

```tex
predictor:  u_i^* = u_i^n - sigma (u_{i+1}^n - u_i^n),
```

```tex
corrector:  u_i^{n+1} =
1/2 (u_i^n + u_i^* - sigma (u_i^* - u_{i-1}^*)).
```

Смысл: сначала используется разность вперед, затем назад, и результат усредняется. На гладких решениях схема имеет второй порядок точности.

Плюсы:

- лучше сохраняет форму гладкого профиля;
- обычно точнее CIR на гладких участках.

Минус:

- около разрывов может давать осцилляции: небольшие выбросы выше и ниже правильного значения.

Это типичная проблема схем высокого порядка: они точнее в гладких областях, но хуже ведут себя около скачков.

### Гибридная схема Федоренко

Идея гибридной схемы: использовать более точную схему там, где решение гладкое, и более устойчивую монотонную схему там, где есть разрыв или резкий излом.

В коде это сделано через ограничитель. Если локально профиль выглядит гладким и монотонным, включается поправка второго порядка. Если обнаруживается резкое изменение, поправка выключается, и схема становится ближе к CIR.

То есть схема пытается объединить два свойства:

- не размазывать гладкие участки слишком сильно;
- не создавать сильные осцилляции около разрывов.

### TVD-схема

TVD означает Total Variation Diminishing, то есть схема с невозрастающей полной вариацией. Полная вариация примерно измеряет суммарную "изрезанность" графика:

```tex
TV(u) = sum_i |u_{i+1} - u_i|.
```

Если схема TVD, она не должна создавать новые лишние экстремумы. Это важно около разрывов: схема не должна превращать один скачок в набор колебаний.

В отчете использован ограничитель Ван Лира:

```tex
phi(r) = (r + |r|) / (1 + |r|).
```

Ограничитель решает, насколько можно включить поправку второго порядка. Если решение гладкое, поправка почти полностью работает. Если есть разрыв, поправка ограничивается.

### Начальные профили

Использованы два профиля.

Прямоугольный импульс:

- имеет два разрыва;
- сложен для численных схем;
- на нем хорошо видно размытие и осцилляции.

Треугольный импульс:

- непрерывен;
- имеет изломы в вершине и на краях;
- обычно дает лучшую сходимость, чем прямоугольник.

### Ошибки и сходимость

В отчете считаются нормы ошибок:

```tex
L1 = h sum_i |u_i - u_i^{exact}|,
```

```tex
L2 = sqrt(h sum_i (u_i - u_i^{exact})^2),
```

```tex
Linf = max_i |u_i - u_i^{exact}|.
```

Также считается экспериментальный порядок по `L1`:

```tex
p = log_2(error_h / error_{h/2}).
```

Если при уменьшении шага в два раза ошибка уменьшается примерно в два раза, то `p ≈ 1`. Если ошибка уменьшается примерно в четыре раза, то `p ≈ 2`.

Важно: на разрывном прямоугольном импульсе даже схемы второго порядка не показывают идеальный второй порядок. Причина не в ошибке реализации, а в математике: порядок схемы второго порядка гарантируется для достаточно гладких решений, а прямоугольник не является гладкой функцией.

## 2. Двумерное нестационарное уравнение теплопроводности

### Постановка

Решается задача

```tex
u_t = u_xx + u_yy,
```

на квадрате

```tex
0 <= x <= 1,
0 <= y <= 1.
```

Граничные условия:

```tex
u(t,0,y) = 0,
u(t,x,0) = 2,
u(t,1,y) = 1,
u(t,x,1) = 3.
```

Физический смысл: есть квадратная пластина. На каждой стороне поддерживается постоянная температура. Нужно понять, как распределение температуры внутри пластины меняется со временем.

В начальный момент внутри пластины в отчете взято нулевое значение, а на границах сразу поставлены заданные температуры.

### Что происходит с решением

Уравнение теплопроводности сглаживает решение. Если где-то есть резкие перепады, со временем они становятся плавнее. В отличие от уравнения переноса, теплопроводность не переносит профиль без изменения формы, а именно "размазывает" температуру из-за диффузии.

При больших временах решение стремилось бы к стационарному распределению, которое решает уравнение Лапласа с теми же границами. В отчете рассматривается конечный момент времени `t = 0.05`, поэтому решение еще не обязано быть стационарным.

### Граничные условия в углах

В углах возникает неоднозначность. Например, в точке `(0,0)` одновременно встречаются левая граница `u=0` и нижняя граница `u=2`. Обе сразу выполнить невозможно.

В отчете угловое значение задается как среднее двух соседних сторон:

```tex
u(0,0) = (0 + 2) / 2 = 1.
```

Такой прием нужен только для численной сетки. Он не меняет смысл задачи внутри области, но делает значения в углах определенными.

### Разностный оператор Лапласа

Вторая производная по `x` заменяется разностью

```tex
u_xx ≈ (u_{i-1,j} - 2u_{i,j} + u_{i+1,j}) / h_x^2.
```

Вторая производная по `y`:

```tex
u_yy ≈ (u_{i,j-1} - 2u_{i,j} + u_{i,j+1}) / h_y^2.
```

Их сумма называется разностным оператором Лапласа.

### Схема расщепления

В задании предложена схема:

```tex
(\tilde u - u^n) / tau = Lambda_x \tilde u,
```

```tex
(u^{n+1} - \tilde u) / tau = Lambda_y u^{n+1}.
```

Это локально-одномерное расщепление. Один двумерный шаг заменяется двумя одномерными:

1. сначала учитывается теплопроводность только по направлению `x`;
2. потом учитывается теплопроводность только по направлению `y`.

Почему это удобно: полностью неявная двумерная схема дала бы большую систему линейных уравнений по всем внутренним узлам сразу. А схема расщепления дает набор маленьких трехдиагональных систем. Такие системы решаются быстро.

### Как получается трехдиагональная система

На первом полушаге для каждого фиксированного `y_j` решаются неизвестные значения вдоль строки по `x`:

```tex
-r u_{i-1,j}^{*} + (1 + 2r)u_{i,j}^{*} - r u_{i+1,j}^{*}
= u_{i,j}^{n},
```

где

```tex
r = tau / h^2.
```

Это трехдиагональная система, потому что в каждом уравнении есть только левый сосед, текущий узел и правый сосед.

На втором полушаге аналогичная система решается по столбцам, то есть по направлению `y`.

### Что показано в ноутбуке

Для этой задачи в отчете построены:

- цветовая карта `u(x,y,t)`;
- трехмерная поверхность;
- сечения `u(x)` при нескольких фиксированных `y`;
- сечения `u(y)` при нескольких фиксированных `x`;
- таблица значений в контрольных точках;
- таблица сходимости при сравнении с сеткой `N = 160`.

Контрольные точки нужны, чтобы не только смотреть на картинку, но и видеть численно, что значения стабилизируются при измельчении сетки.

### Почему шаг времени связан с шагом сетки

В отчете шаг времени выбирается пропорционально `h^2`. Это сделано, чтобы при уменьшении сеточного шага ошибка по времени тоже уменьшалась и не мешала анализировать сходимость по пространству.

Если оставить `tau` слишком большим, то при измельчении пространственной сетки можно получить ситуацию, где пространственная ошибка уже маленькая, а ошибка по времени остается заметной.

## 3. Стационарная задача теплопроводности

### Постановка

Решается уравнение Лапласа

```tex
u_xx + u_yy = 0
```

в квадрате

```tex
0 <= x <= L,
0 <= y <= L,
L = 0.1 м.
```

Граничные условия:

- сторона `AB`, нижняя: `u = 1`;
- сторона `BC`, правая: `u = 2`;
- сторона `CD`, верхняя: `u = 3`;
- сторона `DA`, левая: `u = 4`.

Физический смысл: пластина уже пришла в равновесие. Температура больше не меняется со временем, поэтому нет производной `u_t`. Внутри нет источников тепла, поэтому сумма вторых производных равна нулю.

### Свойство решения

Уравнение Лапласа имеет важное свойство: значение внутри области является усреднением окружающих значений. В разностном виде это прямо видно:

```tex
u_{i,j} = 1/4 (u_{i-1,j} + u_{i+1,j} + u_{i,j-1} + u_{i,j+1}).
```

То есть температура во внутреннем узле равна среднему арифметическому четырех соседей.

Это и лежит в основе итерационных методов.

### Метод Якоби

Метод Якоби обновляет все внутренние узлы по старому слою значений:

```tex
u_{i,j}^{k+1} =
1/4 (u_{i-1,j}^{k} + u_{i+1,j}^{k} + u_{i,j-1}^{k} + u_{i,j+1}^{k}).
```

Плюс:

- очень простой метод.

Минус:

- сходится медленно, потому что на каждой итерации использует только старую информацию.

В коде метод останавливается, когда максимальное изменение за итерацию становится меньше заданного допуска.

### Метод Зейделя

Метод Зейделя похож на Якоби, но как только новое значение узла посчитано, оно сразу используется для следующих узлов:

```tex
u_{i,j}^{new} =
1/4 (u_{i-1,j}^{new} + u_{i+1,j}^{old}
     + u_{i,j-1}^{new} + u_{i,j+1}^{old}).
```

Из-за этого информация по сетке распространяется быстрее. Поэтому метод Зейделя обычно требует меньше итераций, чем Якоби.

### Метод верхней релаксации SOR

SOR означает Successive Over-Relaxation. Это ускоренная версия метода Зейделя.

Сначала считается обычное значение Зейделя:

```tex
u_{GS}.
```

Потом новое значение берется не ровно `u_GS`, а с релаксацией:

```tex
u^{new} = u^{old} + omega (u_{GS} - u^{old}).
```

Если `omega = 1`, это обычный метод Зейделя. Если `1 < omega < 2`, метод делает более смелый шаг в сторону нового значения. При хорошем `omega` это сильно ускоряет сходимость.

В отчете используется теоретически оптимальный параметр:

```tex
rho_J = cos(pi / N),
```

```tex
omega_opt = 2 / (1 + sqrt(1 - rho_J^2)).
```

Здесь `N` связано с числом разбиений стороны квадрата.

### Теоретическое сравнение скоростей

Для квадратной сетки спектральные оценки такие:

```tex
rho_J = cos(pi / N),
```

```tex
rho_GS = rho_J^2.
```

Чем меньше спектральный радиус, тем быстрее убывает ошибка. У метода Зейделя радиус примерно квадрат радиуса Якоби, поэтому он быстрее.

Для SOR при оптимальном `omega` радиус еще меньше, поэтому число итераций резко падает. Это видно в таблице отчета: Якоби требует больше всего итераций, Зейдель меньше, SOR намного меньше.

### Изолинии и сечения

Для стационарной задачи в отчете построены:

- изолинии `T(x,y) = const`;
- цветовая карта температуры;
- графики `T(x)` при разных фиксированных `y`;
- графики `T(y)` при разных фиксированных `x`.

Изолинии показывают линии одинаковой температуры. Если изолинии расположены густо, значит в этой области большой градиент температуры.

Сечения нужны, чтобы увидеть одномерный профиль температуры внутри двумерной области.

### Проверка сходимости по сетке

Для сеток `N = 20, 40, 80, 160` считается решение. Потом более грубые сетки сравниваются с самой мелкой сеткой `N = 160`.

Идея такая: точное аналитическое решение для этой конкретной постановки в отчете не используется, поэтому решение на самой мелкой сетке берется как численный эталон.

Если при увеличении `N` разница с эталоном уменьшается, значит сеточное решение сходится.

## Как читать результаты в ноутбуке

### Для уравнения переноса

Смотри на две вещи:

- насколько график совпадает с точным решением;
- как уменьшаются ошибки при переходе `N = 80 -> 160 -> 320 -> 640`.

Ожидаемая картина:

- CIR самый монотонный, но самый размазанный;
- Мак-Кормак точнее на гладких участках, но может колебаться около скачков;
- Федоренко и TVD дают лучший компромисс;
- TVD обычно особенно хорошо ведет себя на разрывном прямоугольном импульсе.

### Для нестационарной теплопроводности

Смотри:

- поле температуры должно быть гладким;
- значения внутри должны находиться в разумном диапазоне между граничными температурами;
- контрольные значения при увеличении `N` должны стабилизироваться;
- ошибки относительно `N = 160` должны уменьшаться.

### Для стационарной теплопроводности

Смотри:

- решение должно быть гладким внутри квадрата;
- температура около каждой стороны должна приближаться к температуре этой стороны;
- SOR должен требовать сильно меньше итераций, чем Якоби и Зейдель;
- при измельчении сетки значения в контрольных точках должны стабилизироваться.

## Что важно сказать на защите

1. В первой задаче точное решение известно, потому что уравнение переноса просто сдвигает начальный профиль. Это позволяет напрямую считать ошибки.

2. Разрывные решения ухудшают видимый порядок сходимости. Поэтому на прямоугольном импульсе не стоит ждать идеального второго порядка даже от схем второго порядка.

3. TVD и гибридные схемы нужны для баланса между точностью и монотонностью: они стараются быть второго порядка на гладких участках и не создавать лишних осцилляций около разрывов.

4. Во второй задаче схема расщепления удобна тем, что двумерная задача заменяется двумя сериями одномерных трехдиагональных систем.

5. Уравнение теплопроводности сглаживает распределение температуры, поэтому графики должны быть гладкими, без осцилляций.

6. В третьей задаче стационарное решение удовлетворяет принципу среднего: внутренний узел равен среднему своих четырех соседей.

7. Метод Якоби использует только старые значения, поэтому медленный. Метод Зейделя сразу использует новые значения, поэтому быстрее. SOR дополнительно ускоряет процесс за счет параметра релаксации.

8. Сходимость по сетке проверяется сравнением решений на последовательных сетках или с самым мелким расчетом. Если разность уменьшается при уменьшении `h`, численное решение считается сходящимся.

Из корня проекта:

```bash
jupyter nbconvert --to notebook --execute PDE/pde_lab_report.ipynb --output pde_lab_report.ipynb
```
